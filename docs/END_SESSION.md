# End Session — Wrap Up

> Скинь этот файл в конце сессии. AI должен пройтись по чек-листу и обновить артефакты так, чтобы следующая сессия стартовала без потери контекста.

---

## Что должен сделать AI

### 1. Сводка по сессии
Кратко (5-10 пунктов максимум) перечислить:
- **Что было сделано** — фичи, фиксы, рефакторы
- **Какие файлы изменены / созданы / удалены** — список с абсолютными путями
- **Что осталось незавершённым** — если что-то начато но не докручено

### 2. Обновить `docs/WEB_ARCHITECTURE.md`
Если за сессию изменилось что-то из:
- Новые страницы / API routes → добавить в дерево структуры
- Новые ключевые компоненты или хелперы → добавить в раздел "Ключевые компоненты"
- Новые интеграции / зависимости → обновить раздел "Стек" и "Интеграции"
- Изменились инварианты (роли, авторизация, источники данных) → обновить соответствующий раздел
- Закрыт технический долг → убрать из раздела "Известные технические долги"
- Появился новый технический долг → добавить туда же

**Правило:** не дублировать — если фича из ARCHITECTURE.md (Python pipeline) — туда, если из веба — в `WEB_ARCHITECTURE.md`.

### 3. Обновить `progress.txt` (создать если нет)
Записать **в начало файла** (newest first):

```
[YYYY-MM-DD] Краткое название сессии
- Что сделали (1-3 буллета)
- Что не доделали / что дальше
- Важные решения / гипотезы / контекст которые повлияют на следующие сессии
```

Файл живёт в `/Users/alibek/Documents/UFC Predictor/progress.txt`. Цель — за 30 секунд понять "на чём остановились".

Держать максимум **последние 10-15 сессий** — старые удалять.

### 4. Финальная проверка
- `cd web && npx tsc --noEmit -p .` — TypeScript clean
- Если есть тесты — прогнать
- Никаких hanging background процессов (python скрипты, dev серверы которые не должны крутиться)
- Никаких лишних файлов на диске (debug-скрипты, временные .json, недописанные .md)

### 5. Открытые вопросы для пользователя
Если в процессе всплыло что-то требующее моего решения — собрать в финальный список:
- Технические развилки которые я отложил
- Дизайн-решения требующие подтверждения
- Конфликтующие требования

## Чего НЕ делать в конце сессии

- ❌ Не создавать новые .md файлы для каждой мелочи — обновлять существующие
- ❌ Не писать в `progress.txt` больше 5 строк на сессию
- ❌ Не пушить незавершённый код в git без явного запроса
- ❌ Не оставлять `console.log` / debug-комментарии в коде
- ❌ Не оставлять закомментированный код "на потом"
